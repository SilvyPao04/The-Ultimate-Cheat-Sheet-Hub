HTML: HyperText Markup Language

HTML (HyperText Markup Language) is the most basic building block of the Web. It defines the meaning and structure of web content.

HTML Elements

Tag Name Description
<a> Anchor Creates a hyperlink to web pages, files, email addresses, locations in the same page, or anything else a URL can address
<article> Article Represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable
<audio> Audio Used to embed sound content in documents
<br> Line Break Produces a line break in text. It is useful for writing a poem or an address, where the division of lines is significant
<button> Button Interactive element activated by a user with a mouse, keyboard, finger, voice command, or other assistive technology. Once activated, it then performs an action
<details> Details disclosure element Creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state
<figcaption> Figure Caption Represents a caption or legend describing the rest of the contents of its parent <figure> element, providing the <figure> an accessible description.
<form> Form Represents a document section containing interactive controls for submitting information
<img> Image Embeds an image into the document
<input> HTML Input Used to create interactive controls for web-based forms in order to accept data from the user; a wide variety of types of input data and control widgets are available, depending on the device and user agent
<li> List Item Used to represent an item in a list. It must be contained in a parent element: an ordered list (<ol>), an unordered list (<ul>), or a menu (<menu>)
<link> External Resource Link Specifies relationships between the current document and an external resource.
<nav> Navigation Section Represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.
<ol> The Ordered List element Represents an ordered list of items ‚Äî typically rendered as a numbered list.
<option> HTML Option element Represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents
<output> The Output element It is a container element into which a site or app can inject the results of a calculation or the outcome of a user action.
<p> The Paragraph element It represents a paragraph. In HTML paragraphs can be any structural grouping of related content, such as images or form fields.
<pre> The Preformatted Text element Represents preformatted text which is to be presented exactly as written in the HTML file. The text is typically rendered using a non-proportional, or monospaced, font. Whitespace inside this element is displayed as written.
<progress> The Progress Indicator element Displays an indicator showing the completion progress of a task, typically displayed as a progress bar.
<s> The Strikethrough element Renders text with a strikethrough, or a line through it. Use the <s> element to represent things that are no longer relevant or no longer accurate.
<samp> The Sample Output element Used to enclose inline text which represents sample (or quoted) output from a computer program.
<script> The Script element Used to embed executable code or data. This is typically used to embed or refer to JavaScript code.
<search> The generic search element It is a container representing the parts of the document or application with form controls or other content related to performing a search or filtering operation.
<section> The Generic Section element Represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it. Sections should always have a heading, with very few exceptions.
<select> The HTML Select element Represents a control that provides a menu of options. Each <option> element should have a value attribute containing the data value to submit to the server when that option is selected.
<source> The Media or Image Source element Specifies one or more media resources for the <picture>, <audio>, and <video> elements. It is a void element, which means that it has no content and does not require a closing tag.
<strong> The Strong Importance element Indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.
<sub> The Subscript element Specifies inline text which should be displayed as subscript for solely typographical reasons. Subscripts are typically rendered with a lowered baseline using smaller text.
<sup> The Superscript element Specifies inline text which is to be displayed as superscript for solely typographical reasons. Superscripts are usually rendered with a raised baseline using smaller text.
<table> Table The wrapper element for all HTML tables
<tbody> Table Body The set of rows containing actual table data
<td> The Table Data Cell element Defines a cell of a table that contains data and may be used as a child of the <tr> element.
<textarea> The Textarea element Represents a multi-line plain-text editing control, useful when you want to allow users to enter a sizeable amount of free-form text, for example a comment on a review or feedback form.
<tfoot> Table Foot The set of rows defining the footer in a table
<thead> Table Head The set of rows defining the column headers in a table
<time> The (Date) Time element Represents a specific period in time. It may include the datetime attribute to translate dates into machine-readable format, allowing for better search engine results or custom features such as reminders.
<title> The Document Title element Defines the document's title that is shown in a browser's title bar or a page's tab. It only contains text; tags within the element are ignored.
<tr> Table Row The table row container
<track> The Embed Text Track element Used as a child of the media elements, <audio> and <video>. Each track element lets you specify a timed text track (or time-based data) that can be displayed in parallel with the media element, for example to overlay subtitles or closed captions on top of a video or alongside audio tracks.
<ul> The Unordered List element Represents an unordered list of items, typically rendered as a bulleted list.
<video> The Video Embed element Embeds a media player which supports video playback into the document.

Input Types

The <input> element is one of the most powerful and complex in all of HTML due to the sheer number of combinations of input types and attributes.
How an <input> works varies considerably depending on the value of its type attribute.

Tag Type Description
<input type="button"> button Are rendered as simple push buttons, which can be programmed to control custom functionality anywhere on a webpage as required when assigned an event handler function (typically for the click event).
<input type="checkbox"> checkbox Are rendered by default as boxes that are checked (ticked) when activated. The exact appearance depends upon the operating system configuration under which the browser is running.
<input type="color"> color Provide a user interface element that lets a user specify a color, either by using a visual color picker interface or by entering the color into a text field in #rrggbb hexadecimal format.
<input type="date"> date Create input fields that let the user enter a date. The appearance of the date picker input UI varies based on the browser and operating system. The value is normalized to the format yyyy-mm-dd.
<input type="datetime-local"> datetime-local Create input controls that let the user easily enter both a date and a time, including the year, month, and day as well as the time in hours and minutes.
<input type="email"> email Used to let the user enter and edit an email address, or, if the multiple attribute is specified, a list of email addresses.
<input type="file"> file Let the user choose one or more files from their device storage.
<input type="image"> image Used to create graphical submit buttons, i.e. submit buttons that take the form of an image rather than text.
<input type="month"> month Create input fields that let the user enter a month and year allowing a month and year to be easily entered.
<input type="number"> number Used to let the user enter a number. They include built-in validation to reject non-numerical entries.
<input type="password"> password Provide a way for the user to securely enter a password.
<input type="radio"> radio Generally used in radio groups‚Äîcollections of radio buttons describing a set of related options. Only one radio button in a given group can be selected at the same time. Radio buttons are typically rendered as small circles, which are filled or highlighted when selected.
<input type="range"> range Let the user specify a numeric value which must be no less than a given value, and no more than another given value. The precise value, however, is not considered important. This is typically represented using a slider or dial control rather than a text entry box like the number input type.
<input type="reset"> reset Are rendered as buttons, with a default click event handler that resets all inputs in the form to their initial values.
<input type="search"> search Text fields designed for the user to enter search queries into. These are functionally identical to text inputs, but may be styled differently by the user agent.
<input type="submit"> submit Are rendered as buttons. When the click event occurs (typically because the user clicked the button), the user agent attempts to submit the form to the server.
<input type="tel"> tel Are used to let the user enter and edit a telephone number.
<input type="text"> text Create basic single-line text fields.
<input type="time"> time Create input fields designed to let the user easily enter a time (hours and minutes, and optionally seconds). The value is always a 24-hour HH:mm or HH:mm:ss formatted time, with leading zeros, regardless of the UI's input format.
<input type="url"> url Used to let the user enter and edit a URL.
<input type="week"> week Create input fields allowing easy entry of a year plus the ISO 8601 week number during that year (i.e., week 1 to 52 or 53)
